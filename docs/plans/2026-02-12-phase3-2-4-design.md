## Phase 3.2: モデル自由選択

### 現状

- `config/features.json` に `default_model` で1プロバイダー1モデル固定
- `config/pricing.json` にはすでに上位モデル（gpt-4o, gemini-2.0-pro）の料金が定義済み
- API は `model` パラメータを受け取れる構造

### 設計

1. **`config/features.json` に `available_models` 追加**
   ```json
   "available_models": {
     "openai": ["gpt-4o-mini", "gpt-4o"],
     "gemini": ["gemini-2.0-flash", "gemini-2.0-pro"],
     "claude": ["claude-sonnet-4-20250514", "claude-opus-4-20250514"]
   }
   ```

2. **pricing.json に Claude Opus 追加**
   ```json
   "claude-opus-4-20250514": { "input_per_1m": 15.00, "output_per_1m": 75.00 }
   ```

3. **UI: モデルセレクター追加**
   - エンジン選択後にモデルドロップダウンを表示
   - デフォルトは `default_model` の値
   - モデル名の横にコスト目安を表示（例: `gpt-4o ($2.50/1M入力)`）

4. **API: model パラメータの明示的伝達**
   - `/api/decide`, `/api/breakdown`, `/api/compare` に model を渡す
   - 比較ページでは各プロバイダーの選択モデルを使用

## Phase 3.3: RAG連携

### 技術選定

| 項目 | 選択 | 理由 |
|------|------|------|
| ベクトルDB | SQLite + sqlite-vec | 既存SQLite基盤を活用、追加DB不要 |
| エンベディング | OpenAI text-embedding-3-small | 低コスト、十分な精度 |
| ドキュメントパーサー | 自前（MD: 分割、PDF: pdf-parse） | 軽量、依存少 |
| チャンクサイズ | 500トークン、100トークンオーバーラップ | バランス良好 |

### アーキテクチャ

```
ドキュメント
  ↓ アップロード
チャンク分割 → エンベディング → SQLite (vec0)
  ↓
判定リクエスト時:
  タスク内容 → エンベディング → 類似チャンク検索 (top-k=3)
  → プロンプトに関連コンテキスト注入
```

### ファイル構造

```
src/lib/rag/
  ├── chunker.ts        # ドキュメント分割
  ├── embedder.ts       # OpenAI Embeddings API
  ├── vector-store.ts   # sqlite-vec CRUD
  └── retriever.ts      # 類似検索 + コンテキスト構築
config/
  └── rag.json          # チャンクサイズ、top-k 等の設定
```

## Phase 3.4: Web Push通知

### 技術選定

| 項目 | 選択 | 理由 |
|------|------|------|
| Push API | Web Push (VAPID) | 標準API、サーバー依存なし |
| ライブラリ | web-push | Node.js 標準ライブラリ |
| スケジュール | Cron的なAPI Route | Next.js の Route Handler で実現 |

### 機能

1. **定期リマインダー**: 「今日のタスクを決めましょう」（毎朝）
2. **予算アラート**: 月間予算80%超過時に通知
