# Phase 6: チャット風ダッシュボード（羅針盤ファースト）設計書

## Context

ダッシュボードの課題:
- いきなりフォームが表示され「何からやればいいか分からない」
- 各項目（エネルギー、AIエンジン等）の意味が不明
- 羅針盤（目標・夢）が別ページに隔離され、タスク判定との繋がりが薄い

## Design: チャット風ステップバイステップUI

AIが対話形式で1つずつ案内。羅針盤を土台に据え、目標を意識した上でタスク判定を行う。

### 基本方針

- **羅針盤ファースト**: 初回は夢・目標の登録から。リピーターは毎回羅針盤を表示
- **チャット風UI**: AIメッセージ → ユーザー入力 → 次のAIメッセージ、の繰り返し
- **丁寧・落ち着きトーン**: 「こんにちは。今日取り組みたいタスクを教えてください。」
- **詳細設定は折りたたみ**: エンジン・モデル選択は普段隠す
- **段階的開示**: 入力するたびに次のステップが出現（アニメーション付き）

## フロー詳細

### A. 初回ユーザー（羅針盤データなし）

#### Step 1: 羅針盤登録の案内

```
🤖 はじめまして。AI意思決定アシスタントです。

   あなたの目標や夢を教えてください。
   「将来やりたいこと」「ワクワクすること」など、
   何でも構いません。

   これを「羅針盤」として、日々のタスク選びの
   基準にします。
```

入力UI:
- テキスト入力欄 + 「追加」ボタン
- placeholder: 「例: 海外で暮らしたい」
- 追加されたアイテムはチップ（タグ）で表示
- 「スキップして先にタスク判定する」リンクも用意（強制しない）
- 「写真やURLも登録できます → 羅針盤ページへ」リンク

#### Step 2: 登録完了 → タスク入力へ

```
🤖 素敵ですね。羅針盤に登録しました。🧭

   では、今日取り組みたいタスクを教えてください。
   あなたの目標に沿った優先順位を提案します。
```

（以降は B. リピートユーザー の Step 2 と同じ）

---

### B. リピートユーザー（羅針盤データあり）

#### Step 1: 羅針盤の表示（読み取り専用）+ タスク入力

羅針盤は**表示するだけ**。毎回の入力は求めない。
AIがバックグラウンドで判定に使う「土台」として見せる。
追加・編集したい場合のみ「編集」リンクから羅針盤ページへ遷移。

```
🤖 こんにちは。

   🧭 あなたの羅針盤:
   海外で暮らしたい / 英語を話せるようになる / 副業で月10万稼ぐ
   [編集]

   今日取り組みたいタスクを教えてください。
   目標に沿った優先順位を提案します。
```

入力UI:
- テキスト入力欄 + 「追加」ボタン
- placeholder: 「例: 企画書の作成」
- 追加されたタスクはチップで表示（×で削除可）

#### Step 2: 時間選択

タスクが1つ以上追加されると出現（アニメーション）:

```
🤖 ありがとうございます。{N}つのタスクですね。
   今日はどのくらい時間が取れそうですか？
```

選択UI:
- プリセットボタン: [30分] [1時間] [2時間] [3時間+]
- カスタム入力: テキストフィールド（分単位）

#### Step 3: エネルギーレベル

時間を選択すると出現:

```
🤖 今の気力・体力はどのくらいですか？
   低いときは無理のないタスクを提案します。
```

選択UI（横並びボタン）:
- [1 😩 ぐったり]
- [2 😐 まあまあ]
- [3 🙂 普通]
- [4 😊 元気]
- [5 🔥 やる気MAX]

#### Step 4: 確認 + 実行

エネルギーを選択すると出現:

```
🤖 準備完了です。

   📋 タスク: 企画書の作成、メール返信、経費精算
   ⏰ 時間: 1時間
   ⚡ エネルギー: 3 🙂 普通

   [✨ 最適なタスクを判断する]

   ⚙️ 詳細設定 ▼
```

詳細設定（折りたたみ）:
- AIエンジン選択: openai / gemini / claude
- モデル選択: ドロップダウン
- 自動フォールバック: チェックボックス

#### Step 5: 結果表示

```
🤖 分析しました。

   （既存の DecisionResult コンポーネントを表示）
   （羅針盤との関連度も表示）
```

## コンポーネント設計

### 新規コンポーネント

| コンポーネント | 責務 |
|--------------|------|
| `ChatDashboard` | チャット風UIのコンテナ。ステップの表示/非表示を管理 |
| `ChatMessage` | AIメッセージの吹き出し（🤖アイコン + テキスト） |
| `CompassSummary` | 羅針盤のコンパクト表示（チップ形式 + 編集リンク） |
| `TaskChipInput` | タスク入力 + チップ表示（追加/削除） |
| `TimeSelector` | 時間プリセットボタン + カスタム入力 |
| `EnergySelector` | エネルギーレベルボタン（絵文字付き） |
| `AdvancedSettings` | 折りたたみ式の詳細設定（エンジン/モデル） |

### 既存コンポーネントの再利用

| コンポーネント | 変更 |
|--------------|------|
| `DecisionResult` | 変更なし。Step 5 でそのまま表示 |
| `BreakdownResult` | 変更なし |

### 状態管理

```typescript
type ChatStep = "compass-setup" | "tasks" | "time" | "energy" | "confirm" | "loading" | "result";

interface ChatState {
  step: ChatStep;
  compassItems: { title: string }[];   // APIから取得
  tasks: string[];
  availableTime: number;
  energyLevel: number;
  // 詳細設定
  provider: string;
  model: string;
  autoFallback: boolean;
}
```

### アニメーション

- 各ステップの出現: `transition-all duration-500 ease-out` + `opacity-0 → 1` + `translate-y-4 → 0`
- スクロール: 新ステップ出現時に自動スクロール（`scrollIntoView({ behavior: "smooth" })`）

## 羅針盤の簡易登録（ダッシュボード内）

初回の羅針盤登録はダッシュボード内で完結させる。

### API呼び出し

```
POST /api/compass
{ type: "text", title: "海外で暮らしたい", content: "海外で暮らしたい" }
```

既存の Compass API をそのまま使用。テキスト形式のみ対応（URLや画像は羅針盤ページへ誘導）。

### 取得

```
GET /api/compass
→ items[] の title を表示
```
